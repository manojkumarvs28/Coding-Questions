	createNamedQuery and createNativeQuery
	  1. createNativeQuery
	    Query updateThemeOrderQuery = em.createNativeQuery("UPDATE T_THEME SET THEME_ORDER = ?1 WHERE TECHNICAL_NAME = ?2");
        updateThemeOrderQuery.setParameter(1, themeOrder);
        updateThemeOrderQuery.setParameter(2, themeTechnicalName);
        updateThemeOrderQuery.executeUpdate();

		//Here numbers are specified in query and numbers are substitued in setParameter
		//In nativeQuery, QUERY is mentioned in Capitals particularly the table Name -> T_THEME
		// Note: When update or delete query we use "executeUpdate". In case of select we use "getResultList" or "getSingleResultList". This is
		// observed in both the cases


	 2. createNamedQuery
	      Query query = manager.createNamedQuery("SELECT r from TThemeRelation r WHERE r.themeId = :themeId  ORDER BY r.themeCPRelOrder", TThemeRelation.class);
            query.setParameter("themeId", themeIdentifier);
            List<TThemeRelation> relations = query.getResultList();
		// Here query and table name is specified in smaller
	    //Here ":themeId" is specified instead of numbers in Query and it is replaced in setParameter
		//"TThemeRelation.class" if specified the getResultlist will give objects of type "TThemeRelation" otherwise it will return
		// List<Object>[]. This is applicable in both cases.
		//If class is specified as in above case then the query becomes TypedQuery. In this case it will be TypedQuery<TThemeRelation>
